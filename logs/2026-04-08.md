### 失敗したこと
- SiteStatus vs ProcessStatus の型不一致（既存コードのgetStatusColor/Labelが ProcessStatusのみ対応だった）
- Base UIベースのshadcn/uiにasChildプロパティなし（Radix UI前提で書いてしまった）
- turbopack.root を相対パス"."で指定して警告（__dirnameで解消）

### うまくいったこと
- Next.js 16の async params パターンが正しく動作
- CSS Grid ガントチャートが外部ライブラリなしで実装できた
- NDJSON streaming でAI応答のプログレッシブ表示が実現
- APIキー未設定時のモックデータフォールバックが営業デモとして十分な品質

### 解決方法
- SiteStatus対応: utils.tsの関数シグネチャをunion型に変更 + mapにscheduled/suspended追加
- asChild: TooltipTriggerに直接classNameとstyleを渡す形に変更
- turbopack.root: __dirname使用で絶対パス指定

### 学んだこと・知見
- shadcn/ui v4は@base-ui/reactベースでRadix UIのasChildパターンが使えない
- Next.js 16のturbopack.rootは絶対パス必須
- WeatherForecast型をReturnTypeで取ろうとすると"does not satisfy constraint '(...args: any) => any'"エラー（型はオブジェクトなので直接参照すべき）
