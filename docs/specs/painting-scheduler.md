# 塗装工程AIスケジューラー デモアプリ 仕様書

- **ステータス**: 確定
- **作成日**: 2026-04-07
- **対象企業**: 株式会社KYOEI（大阪府高石市・外壁塗り替え専門店）
- **目的**: エンジニア講座の営業デモとして、塗装業界の課題を解決するAIアプリを実演する

---

## 概要

### 何を実装するか
天気予報と連動して塗装工事の工程表を自動最適化するWebアプリケーション。AIが天気データを分析し、雨天による工程変更を事前にシミュレーション・提案する。

### なぜ必要か
塗装業界では天候（特に雨）によって作業工程が大きく左右される。現状は人手でスケジュール調整しており、急な天候変化への対応が課題。AIによる自動化で、工期短縮とコスト削減を実現できることをデモで示す。

## 画面構成

### 1. ダッシュボード（トップページ）
- **現場一覧カード**: 各現場の名前・住所・進捗率・ステータスを一覧表示
- **天気予報ウィジェット**: 今後1〜2週間の天気をアイコン付きで表示
- **天気アラートバナー**: 「○日後に雨予報。○○現場の△△工程に影響あり」を目立つUI で通知
- **天気データ切り替えスイッチ**: リアルAPI ⇔ デモ用モックの切り替え
- **複数現場の同時管理**: 現在稼働中の全現場のステータスを俯瞰

### 2. 現場詳細ページ
- **物件情報**: 物件名・住所・施主名・建物種別（戸建て/マンション/公共施設）・塗装面積
- **工程表（ガントチャート風）**: 各工程をタイムラインで横軸表示
- **工程ステータス**: 未着手 / 進行中 / 完了 / 天候待ち
- **コスト・工期影響表示**: 天候による延長日数と影響をリアルタイム表示

### 3. AIシミュレーション画面
- **事前シミュレーション**: 天気予報を元に最適な工程スケジュールを自動生成
- **緊急リスケ**: 急な天候変化に対する代替スケジュールの即時生成
- **ビフォーアフター比較**: 変更前→変更後の工程表を並べて表示
- **AI提案カード**: AIアイコン付きで「なぜこの変更をしたか」の理由説明

### 4. AI演出UI（全画面共通）
- **思考中アニメーション**: AIが処理中にパルス/タイピング風アニメーション表示
- **ステータスメッセージ**: 「AIが天気データを分析中...」「最適な工程を計算しています...」
- **チャット風理由説明**: 「雨予報のため中塗りを2日後にずらしました」等のAIコメント
- **AIアイコンバッジ**: どの要素がAIの判断によるものかが一目でわかる

### 天候判定ルール
- 気温5℃以上・湿度85%未満が施工条件
- 雨天時は塗装系工程（3〜10）すべて不可
- 足場関連（2,12）は小雨可・強風NG
- 点検（11）は目視可・手直し塗装は不可

## 天気データ

### リアルモード
- OpenWeatherMap API（無料枠: 1,000回/日）
- 16日間予報を取得
- 対象地域: 大阪府高石市（KYOEI所在地）

### デモモード（モックデータ）
- 事前定義されたシナリオを用意:
  - **シナリオ1「順調」**: 2週間晴れ続き → 最短工期で完了
  - **シナリオ2「途中で雨」**: 5日目に3日間雨 → AIが工程組み替え
  - **シナリオ3「梅雨」**: 断続的に雨 → AIが大幅リスケ＆コスト影響表示
- UIのトグルスイッチで切り替え可能

## AIシミュレーション仕様

### 使用AI
- Claude API（Anthropic）
- モデル: claude-sonnet-4-6（コスト・速度バランス）

### 事前シミュレーション
1. 天気予報データ（1〜2週間分）を取得
2. 各工程の天候条件と照合
3. Claude APIに工程データ + 天気データを送信
4. AIが最適なスケジュールを生成
5. 変更理由を自然言語で説明

### 緊急リスケ
1. 天気データの変化を検知（デモではボタン操作でトリガー）
2. 現在の進捗状況と残工程を確認
3. Claude APIに残工程 + 新天気データを送信
4. AIが代替スケジュールを即時生成
5. ビフォーアフターで変更点を表示

### AIへのプロンプト設計
- 塗装業界の専門知識を含むシステムプロンプト
- 天候条件（気温5℃以上・湿度85%未満）のルールを明示
- 工程間の依存関係を考慮
- コスト・工期への影響も算出して報告

## 品質要件

### UI/UX
- モダンで洗練されたデザイン（shadcn/ui ベース）
- スムーズなアニメーション（Framer Motion）
- レスポンシブ対応（PC / タブレット / スマホ）
- ダークモード非対応（明るいプロフェッショナルなテーマ）

### AI演出
- AIが処理中であることが視覚的に明確
- AIの判断根拠が自然言語で表示される
- AIアイコン・バッジで「これはAIの提案」が一目でわかる
- 思考プロセスの段階的な表示（分析中→計算中→提案生成中）

### パフォーマンス
- ページ遷移はスムーズ（App Router活用）
- AI応答は streaming で段階的に表示
- 天気データはキャッシュして不要なAPI呼び出しを削減
